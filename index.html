Default configuration:

This example shows how to use the default configuration over a form.


$('form').flury();

<form>
  <label for="name">Name:</label>
  <input id="name" type="text" /><br />

  <label for="email">E-mail:</label>
  <input id="email" type="email" />
</form>

Including fields:

You can hook just some types of fields changing the attribute include: 

$('form').flury({ include: 'input:text' });

Excluding fields:

The same way you can exclude some fields changing the attribute exclude: 

$('form').flury({ exclude: 'input:password' });

Using direct target instead a form, the include and exclude gets ignored.

Single target:

You can bind just some field without bind the whole form. 

$('#name').flury();

Changing the settings globally:

You can change any option mentioning the scope $.fn.flury.defaults. + option_name. This setup must be called before you bind the plugin. 

$.fn.flury.defaults.include = 'input:text';
$.fn.flury.defaults.exclude = 'input:password';

Options:

include: ':input'

Fields to be hooked.

exclude: 'input[type="button"], input[type="checkbox"], input[type="image"], input[type="radio"], input[type="submit"]'

Fields to be excluded of the hook.

Functions:

$('form').flury('set', { option: 'value' })

Override the options declared.
